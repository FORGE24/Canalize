# Canalize 地形生成文档

## 概览
Canalize 是一个 Minecraft模组，它通过 JNI 使用高性能 C++ 实现完全重写了主世界（Overworld）的地形生成。主要目标是突破原版高度限制（扩展至 **1808 格**）并实现先进的噪声算法以生成真实、科学的地形。

## 核心技术

- **语言**: C++ (原生生成) & Java (桥接/游戏集成)
- **接口**: JNI (Java Native Interface)
- **优化**: AVX2 指令集 (SIMD) 用于加速噪声计算
- **高度限制**: 1808 格 (Min Y: -64, Max Y: 1744)

## 参考文献

1.  **Musgrave, F. K., Kolb, C. E., & Mace, R. S. (1989).** *The synthesis and rendering of eroded fractal terrains*. ACM SIGGRAPH Computer Graphics, 23(3), 41-50. [DOI: 10.1145/74334.74337](https://dl.acm.org/doi/10.1145/74334.74337)
    - 本项目中的 **Ridged FBM** 和 **水力侵蚀模拟** (Hydraulic Erosion) 思想主要参考了该论文。Musgrave 提出的基于噪声绝对值的脊状分形（Ridged Multifractal）是生成山脉骨架的核心算法。

2.  **Inigo Quilez.** *Domain Warping*. [iquilezles.org](https://iquilezles.org/articles/warp/)
    - 本项目中的 **领域扭曲** (Domain Warping) 实现参考了 Inigo Quilez 的文章。通过多级 FBM 对坐标进行偏移 (`f(p + f(p))`)，模拟了流体流动产生的自然畸变，极大增强了地形的真实感。

## 噪声算法

地形生成依赖于 `src/cpp/jni.cpp` 中实现的一套自定义噪声算法栈。

### 1. Simplex 噪声 (OS2S 近似)
我们使用了基于 Ken Perlin 参考实现的 2D Simplex 噪声。这是所有其他噪声函数的基础构建块。
- **原因**: 比经典 Perlin 噪声更快且伪影更少。
- **用途**: 基础地形形状和微观细节。

### 2. 分形布朗运动 (FBM)
标准 FBM 用于叠加多个八度（Octaves）的噪声。
- **孔隙度 (Lacunarity)**: 设置为 `2.01` (而不是精确的 `2.0`) 以避免网格对齐伪影。
- **用途**: "大陆性"（生物群系控制）、海底和平原地
形。

### 3. 脊状 FBM (Ridged FBM)
FBM 的一种变体，其中噪声取绝对值并反转 (`1.0 - abs(n)`)。
- **效果**: 创造出类似山峰的尖锐脊状结构。
- **用途**: 巨型山脉的"脊柱"或骨架结构。

### 4. Worley 噪声 (细胞噪声)
一种计算到最近特征点距离的细胞噪声算法。
- **效果**: 创造出具有明显棱角的"崎岖"或"岩石"纹理。
- **用途**: 应用于山顶，赋予其坚硬、岩石般的外观，打破 Simplex 噪声的平滑感。

### 5. 领域扭曲 (水力侵蚀模拟)
我们实现了"领域扭曲"（灵感来自 Inigo Quilez）来模拟侵蚀。不再是采样 `noise(x, y)`，而是采样 `noise(x + noise(x), y + noise(y))`。
- **实现**: 多级 FBM 扭曲。
- **效果**: 扭曲地形坐标以产生"流动"模式，模拟水流随时间推移雕刻出的河道和山谷。
- **应用**: 深度调制山脉高度图。我们使用 **脊状 FBM** 层作为侵蚀掩码，在保留山脊的同时雕刻出深邃、尖锐的河谷。

## 生成流程

### 步骤 1: 大陆性 (Continentalness)
低频 FBM (频率 ~0.0002) 决定宏观生物群系布局：
- `< -0.2`: 海洋 (Ocean)
- `-0.2 到 0.4`: 平原/丘陵 (Plains/Rolling Hills)
- `> 0.4`: 巨型山脉 (Massive Mountains)

### 步骤 2: 山脉合成
山脉生成是流程中最复杂的部分：
1.  **基础形状**: 使用 **脊状 FBM** 生成尖锐、巨大的基座。
2.  **岩石细节**: 在山顶混合 **Worley 噪声** 以增加"破碎"的岩石细节。
3.  **侵蚀过程**: **领域扭曲** 函数生成侵蚀图。该图用于激进地雕刻地形：
    ```cpp
    mountainShape *= (0.3f + 0.7f * erosion);
    ```
    这在受侵蚀区域仅保留 30% 的原始质量，创造出深邃、戏剧性的山谷。
4.  **高度缩放**: 结果进行指数运算 (`pow(x, 3.5)`) 以产生指数级坡度，然后缩放到目标高度 ~1700 格。

### 步骤 3: 微观细节 (抖动/Dithering)
为了解决将浮点高度转换为整数方块坐标时产生的"阶梯"或"梯田"伪影：
- 在最终高度上叠加一层高频、低振幅的 Simplex 噪声。
- **振幅**: +/- 2.0 格。
- **结果**: 将平坦的整数平面打破为粗糙、自然的地表。

### 步骤 4: 生物群系混合
我们使用 `smoothstep` 插值在生物群系之间（海洋 -> 平原 -> 山脉）进行混合。
- **目标**: 防止不同地形类型之间出现"墙"或硬区块边界。
- **逻辑**: "大陆性"值驱动一个平滑过渡因子，在大范围内混合平原生成器和山脉生成器的高度。

## 技术修复

### 线程安全
- **问题**: 在多线程环境中使用 `srand()`/`rand()` 导致竞态条件和区块生成错误。
- **解决方案**: 将所有随机生成替换为 **无状态哈希函数** 和静态置换表。

### 结构生成
- **问题**: 原版结构（村庄）生成在海洋中，因为生物群系检查与高度检查脱节。
- **解决方案**: `NativeBiomeSource` 现在在高度低于海平面 (`63`) 时严格将生物群系 ID 钳制为 `海洋 (0)`，确保结构位置验证正确。

### 高度限制
- **问题**: 原版硬编码限制 320/384。
- **解决方案**:
    1.  **数据包**: `data/minecraft/dimension_type/overworld.json` 将 `height` 和 `logical_height` 覆盖为 `1808`。
    2.  **Mixin**: `MixinWorldPresets` 注入自定义的 `NativeChunkGenerator`。
