# Canalize 插件管理器 - 实现文档

## 快速开始

### 访问插件管理器
1. 打开游戏选项屏幕（ESC → "选项"）
2. 点击左下方的 **"Canalize 插件管理器"** 按钮
3. 将显示插件管理器主窗口

### 用户界面概览

#### 标签页导航
插件管理器包含6个主要标签页：

| 标签页 | 功能 | 快捷说明 |
|--------|------|---------|
| **插件列表** | 管理所有插件（加载/删除/重载） | 插件发现和基本操作 |
| **插件详情** | 查看选中插件的详细信息 | 依赖、版本、作者等 |
| **配置** | 全局设置和性能调优 | 日志级别、监控等 |
| **统计** | 性能指标和分析数据 | 执行时间、内存占用等 |
| **日志** | 实时日志流和查询 | 过滤、搜索、导出日志 |
| **高级** | 系统信息和诊断工具 | 热插拔、版本检查等 |

## 架构设计

### 类层次结构

```
AbstractPluginTab (抽象基类)
  ├─ PluginsTab (插件列表)
  ├─ DetailTab (插件详情)
  ├─ ConfigTab (配置)
  ├─ StatsTab (统计)
  ├─ LogsTab (日志)
  └─ AdvancedTab (高级)

PluginManagerScreen (主屏幕)
  └─ contains multiple tabs + navigation

ScreenEventHandler (事件处理)
  └─ adds button to OptionsScreen
```

### 事件流

```
1. 用户打开选项屏幕
2. ScreenEventHandler.onScreenOpen() 被触发
3. 添加"Canalize 插件管理器"按钮
4. 用户点击按钮
5. PluginManagerScreen 打开
6. 显示默认的PluginsTab标签页
7. 用户可以切换标签页浏览/配置插件
```

## 功能详细说明

### 插件列表标签页功能
- **显示方式**: 列表形式，每个插件一行
- **可缕操作**: 
  - 双击插件查看详情
  - 右键菜单：启用/禁用、删除、查看日志
  - 拖拽排序优先级（未来功能）
- **搜索过滤**: 按名称、版本、作者搜索

### 日志查看功能
- **实时显示**: 从NativeQueryBridge获取日志
- **颜色编码**:
  - 🔴 ERROR - 红色 (0xFFFF0000)
  - 🟡 WARN - 黄色 (0xFFFFFF00)
  - 🟢 INFO - 绿色 (0xFF00FF00)
  - 🔵 DEBUG - 蓝色 (0xFF0088FF)
- **导出格式**: JSON/CSV/纯文本
- **搜索功能**: 正则表达式匹配

### 统计分析功能
- **实时监控**: 显示当前插件性能指标
- **历史数据**: 保存24小时性能数据
- **图表显示**: 性能趋势曲线
- **优化建议**: 基于数据自动生成优化方案

### 配置管理功能
- **导入/导出**: 支持JSON格式配置备份
- **配置版本**: 自动记录配置历史
- **一键恢复**: 恢复到默认或之前的配置
- **配置验证**: 检查配置完整性和有效性

## 使用示例

### 场景1: 查看插件信息
1. 打开选项 → 点击"Canalize 插件管理器"
2. 在"插件列表"标签页中查看所有已加载插件
3. 点击插件进入"插件详情"标签页查看详细信息

### 场景2: 查看实时日志
1. 点击"日志"标签页
2. 实时日志自动显示
3. 使用"过滤"功能查看特定插件的日志
4. 点击"导出"保存日志文件

### 场景3: 性能优化
1. 点击"统计"标签页查看性能数据
2. 点击"配置"标签页调整监测间隔
3. 根据建议优化配置
4. 点击"导出配置"保存优化方案

### 场景4: 问题诊断
1. 点击"高级"标签页
2. 点击"诊断工具"运行系统诊断
3. 点击"导出诊断报告"生成报告文件
4. 在"日志"标签页中搜索错误日志

## 开发者注意事项

### 扩展标签页
要添加新的标签页，只需：
1. 创建继承 `AbstractPluginTab` 的新类
2. 实现必要的抽象方法
3. 在 `PluginManagerScreen.init()` 中注册新标签页

```java
public class MyNewTab extends AbstractPluginTab {
    public MyNewTab(Screen parent, int tabIndex, String tabName) {
        super(parent, tabIndex, tabName);
    }
    // ... implement abstract methods
}
```

### 与NativeQueryBridge整合
所有数据查询通过NativeQueryBridge获取：
```java
String[] logs = NativeQueryBridge.drainLog(); // 获取日志
Map<String, String> status = NativeQueryBridge.getStatusMap(); // 获取状态
```

## FAQ

**Q: 如何添加更多插件？**
A: 使用PluginManagerScreen中的"加载插件"按钮，或将插件JAR放入指定的插件目录。

**Q: 日志会一直保存吗？**
A: 默认保存最近1000条日志。配置"日志大小限制"可以调整。

**Q: 是否支持插件热重载？**
A: 可以，在"高级"标签页中点击"热插拔"按钮。

**Q: 如何导出性能报告？**
A: 在"统计"标签页中点击"导出"，选择格式（JSON/CSV）。
